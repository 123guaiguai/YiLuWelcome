搭建博客

所谓博客，就是能够随时发表有关自己日常学习生活文章的软件（或网站），比如我们熟知的新浪博客。而这里即将介绍的博客搭建则是基于Web的博客网站，你可以跟随下面的指引建立一个属于自己的博客。

如何搭建博客😃

现在网上有很多搭建博客的框架（或系统），这些框架只需要通过简单的配置就可以快速生成自己想要的博客站点，常见的搭建博客的框架有以下几个：

而我们的这篇教程，主要引导大家如何去用Hexo快速方便搭建属于自己的博客。

Hexo -> 快速，简介，高效的博客框架 📒

正如标题所说，Hexo是一款博客框架，它相比wordPress，上手难度较低。

在正式使用Hexo之前，你需要先了解两样东西：

  1. node / npm

  2. git

Node / Npm

Nodejs是Javascript一个运行环境，你暂时不用去深入了解它，只需要知道：有了Nodejs，Javascript也可以在非浏览器端进行运行，这篇文章是关于Nodejs相关介绍：什么是Nodejs？

至于npm，它是Nodejs的包管理工具，它等同于python中的pip，你可以通过npm下载Node社区一切的第三方开源插件，SDK等等。

在Nodejs官网可以下载Nodejs环境，而npm是跟随Nodejs一起安装的，所以不需要单独下载。

  • 注意：下载时请下载LTS版本，即长时间支持版本（也即稳定版）。

git

关于git的相关介绍和引导在另一篇文章有些，这里不再叙述。传送门🚪

当你了解完前置知识后，🎉恭喜你，你可以正式地开启Hexo之旅了。

Hexo安装

Hexo它本质上就是托管到npm上的，所以你可以使用npm快速地进行安装。

具体的安装流程请参照官网的教程，官网的教程非常详细，这里不再叙述。

Hexo部署

在安装完Hexo后，你可以在本地的4000端口启动一个本地服务，但是它依旧是本地服务，如果想要外网的人看到自己的博客，需要将其托管到服务器上。

不用担心，GitHub Page 可以帮你做这件事，你不需要去自己购买服务器，也不需要去备案域名，只需要将Hexo服务部署到Github Page上，你就拥有一个自己的外网可访问的博客站点了。

这里是有关于Github Page的介绍以及Hexo部署到Github Page的教程。

在做完这一切工作后，🎉恭喜你，你现在拥有了自己的博客站点，你可以去官网查看更多的自定义方案，包括主题选择，博客配置等等。

提交方式

建立一个.txt文件，写上你的博客域名。
